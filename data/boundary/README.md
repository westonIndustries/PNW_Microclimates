# Boundary Reference Data

All files here are generated by running `python scripts/generate_region1_boundary.py`.
Do not edit them manually — regenerate from source if the region definition changes.

---

## region_1.geojson

Single-polygon GeoJSON covering the combined Oregon + Washington study area.

| Property | Value |
|----------|-------|
| `region_code` | `"R1"` |
| `region_name` | `"region_1"` |
| `states` | `["OR", "WA"]` |
| `bounding_box` | `{minx, miny, maxx, maxy}` in EPSG:4326 |

Used by `clip_to_boundary.py` as the authoritative mask for all raster inputs.

## region_registry.csv

**The primary input file that defines regions.** One row per ZIP code. The pipeline groups rows by `region_code` to discover all regions and their constituent ZIP codes. Adding a new region or reassigning ZIP codes is a CSV edit — no code changes needed.

| Column | Type | Description |
|--------|------|-------------|
| `zip_code` | string | 5-digit ZIP code (must match a row in `zipcodes_orwa.geojson`) |
| `state` | string | `"OR"` or `"WA"` |
| `region_code` | string | Region identifier (e.g. `R1`); all ZIPs with the same code form one region |
| `region_name` | string | Human-readable region name (e.g. `region_1`) |
| `base_station` | string | NOAA ICAO station code assigned to this ZIP (e.g. `KPDX`) |
| `lidar_vintage` | int | Year of LiDAR DEM used for this ZIP's area (e.g. `2021`) |

The pipeline loader (`src/loaders/load_region_registry.py`) groups by `region_code` and derives:
- `zip_codes` — list of all ZIPs in the region
- `base_stations` — unique list of stations across those ZIPs
- `bounding_box` — computed from the min/max centroids of the region's ZIPs (padded 0.5°)
- `lidar_vintage` — taken from the rows (must be consistent within a region)

Config constant: `REGION_REGISTRY_CSV` (default: `data/boundary/region_registry.csv`)

## zipcodes_orwa.geojson / zipcodes_orwa.csv

Merged ZIP code boundary layer for OR + WA, built from three sources in priority order.
The `source` field on each feature tells you which dataset the geometry came from.

### Source Priority

| Priority | Source | Coverage | Accuracy |
|----------|--------|----------|----------|
| 1 (highest) | **Oregon Metro RLIS** | Portland metro area | Tax-lot accurate — most precise for the core service area |
| 2 | **OpenDataSoft US ZIP Boundaries** | All OR + WA ZIPs not in RLIS | Better attribute data than Census (includes place names) |
| 3 (fallback) | **Census TIGER/Line ZCTA** | Any remaining ZIPs | Auto-downloaded; covers full national extent |

### Placing Source Files

Drop optional source files into `data/boundary/sources/` before running the script:

```
data/boundary/sources/
├── rlis_zipcodes.geojson        # Oregon Metro RLIS — export from https://rlisdiscovery.oregonmetro.gov
└── opendata_zipcodes_orwa.geojson  # OpenDataSoft — export from https://public.opendatasoft.com/explore/dataset/us-zip-code-territory/
                                    # Filter: state = OR, WA — export as GeoJSON
```

If neither file is present the script falls back entirely to Census TIGER/Line.

### Output Schema

| Column / Property | Description |
|-------------------|-------------|
| `zip_code` | 5-digit ZIP code string |
| `state` | `"OR"` or `"WA"` (spatial join with state boundaries) |
| `po_name` | Post office / place name |
| `region_code` | `"R1"` for all features |
| `source` | `"rlis"`, `"opendata"`, or `"census"` |
| `centroid_lon` | Longitude of polygon centroid (EPSG:4326) — CSV only |
| `centroid_lat` | Latitude of polygon centroid (EPSG:4326) — CSV only |

Config constants: `ZIPCODES_GEOJSON`, `ZIPCODES_CSV`

## Regenerating

```bash
# Minimal — Census TIGER/Line only (auto-downloaded)
python scripts/generate_region1_boundary.py

# With RLIS + OpenDataSoft files already placed in sources/
python scripts/generate_region1_boundary.py

# Use pre-downloaded TIGER/Line shapefiles to avoid re-downloading
python scripts/generate_region1_boundary.py --source-dir data/boundary/sources
```
