# Boundary Reference Data

All files here are generated by running the scripts in `scripts/`:
- `python scripts/generate_zipcodes_orwa.py` — generates ZIP code boundaries
- `python scripts/generate_region1_boundary.py` — generates region boundary and registry
- `python scripts/write_boundary_map.py` — generates interactive map

Do not edit these files manually — regenerate from source if the region definition changes.

---

## region_1.geojson

Single-polygon GeoJSON covering the combined Oregon + Washington study area.

**Format**: GeoJSON FeatureCollection with one Feature

**Properties**:
| Property | Type | Description |
|----------|------|-------------|
| `region_code` | string | `"R1"` — region identifier |
| `region_name` | string | `"region_1"` — human-readable name |
| `states` | string | `"OR,WA"` — comma-separated state abbreviations |
| `bounding_box` | object | `{minx, miny, maxx, maxy}` in EPSG:4326 |

**CRS**: EPSG:4326 (WGS84)

**Usage**: Used by `clip_to_boundary.py` as the authoritative mask for all raster inputs.

## region_registry.csv

**The primary input file that defines regions.** One row per ZIP code. The pipeline groups rows by `region_code` to discover all regions and their constituent ZIP codes. Adding a new region or reassigning ZIP codes is a CSV edit — no code changes needed.

**Format**: CSV with headers

**Columns**:
| Column | Type | Description |
|--------|------|-------------|
| `zip_code` | string | 5-digit ZIP code (must match a row in `zipcodes_orwa.geojson`) |
| `state` | string | `"OR"` or `"WA"` |
| `region_code` | string | Region identifier (e.g. `R1`); all ZIPs with the same code form one region |
| `region_name` | string | Human-readable region name (e.g. `region_1`) |
| `base_station` | string | NOAA ICAO station code assigned to this ZIP (e.g. `KPDX`) |
| `lidar_vintage` | int | Year of LiDAR DEM used for this ZIP's area (e.g. `2021`) |

**Example**:
```
zip_code,state,region_code,region_name,base_station,lidar_vintage
97201,OR,R1,region_1,KPDX,2021
97202,OR,R1,region_1,KPDX,2021
98101,WA,R1,region_1,KSEA,2021
```

**Usage**: The pipeline loader (`src/loaders/load_region_registry.py`) groups by `region_code` and derives:
- `zip_codes` — list of all ZIPs in the region
- `base_stations` — unique list of stations across those ZIPs
- `bounding_box` — computed from the min/max centroids of the region's ZIPs (padded 0.5°)
- `lidar_vintage` — taken from the rows (must be consistent within a region)

**Config constant**: `REGION_REGISTRY_CSV` (default: `data/boundary/region_registry.csv`)

## zipcodes_orwa.geojson / zipcodes_orwa.csv

Merged ZIP code boundary layer for OR + WA, built from three sources in priority order.
The `source` field on each feature tells you which dataset the geometry came from.

### Source Priority

| Priority | Source | Coverage | Accuracy |
|----------|--------|----------|----------|
| 1 (highest) | **Oregon Metro RLIS** | Portland metro area | Tax-lot accurate — most precise for the core service area |
| 2 | **OpenDataSoft US ZIP Boundaries** | All OR + WA ZIPs not in RLIS | Better attribute data than Census (includes place names) |
| 3 (fallback) | **Census TIGER/Line ZCTA** | Any remaining ZIPs | Auto-downloaded; covers full national extent |

### Placing Source Files

Drop optional source files into `data/boundary/sources/` before running the script:

```
data/boundary/sources/
├── rlis_zipcodes.geojson        # Oregon Metro RLIS — export from https://rlisdiscovery.oregonmetro.gov
└── opendata_zipcodes_orwa.geojson  # OpenDataSoft — export from https://public.opendatasoft.com/explore/dataset/us-zip-code-territory/
                                    # Filter: state = OR, WA — export as GeoJSON
```

If neither file is present the script falls back entirely to Census TIGER/Line.

### Output Schema

**GeoJSON** (`zipcodes_orwa.geojson`):
| Property | Type | Description |
|----------|------|-------------|
| `zip_code` | string | 5-digit ZIP code |
| `state` | string | `"OR"` or `"WA"` (spatial join with state boundaries) |
| `po_name` | string | Post office / place name |
| `region_code` | string | `"R1"` for all features |
| `source` | string | `"rlis"`, `"opendata"`, or `"census"` |
| `geometry` | Polygon | ZIP code boundary polygon in EPSG:4326 |

**CSV** (`zipcodes_orwa.csv`):
| Column | Type | Description |
|--------|------|-------------|
| `zip_code` | string | 5-digit ZIP code |
| `state` | string | `"OR"` or `"WA"` |
| `po_name` | string | Post office / place name |
| `region_code` | string | `"R1"` for all rows |
| `centroid_lon` | float | Longitude of polygon centroid (EPSG:4326) |
| `centroid_lat` | float | Latitude of polygon centroid (EPSG:4326) |

**Example CSV row**:
```
97201,OR,Portland,R1,-122.6765,45.5152
```

**Config constants**: `ZIPCODES_GEOJSON`, `ZIPCODES_CSV`

## boundary_map.html

Self-contained interactive Leaflet HTML map showing:
- Region boundary (red dashed line)
- ZIP code boundaries colored by source (RLIS=blue, OpenDataSoft=orange, Census=green)
- Layer controls to toggle region and ZIP code visibility
- Popup information on click

**Generation**:
```bash
python scripts/write_boundary_map.py --output output/boundary_map.html
```

**Usage**: Open in any web browser. No external dependencies required (all CSS/JS inlined or CDN-hosted).

## Regenerating

### Generate ZIP Code Boundaries

```bash
# Minimal — Census TIGER/Line only (auto-downloaded)
python scripts/generate_zipcodes_orwa.py

# With RLIS + OpenDataSoft files already placed in sources/
python scripts/generate_zipcodes_orwa.py --source-dir data/boundary/sources

# Specify custom output directory
python scripts/generate_zipcodes_orwa.py --output-dir data/boundary
```

### Generate Region Boundary and Registry

```bash
# Minimal — Census TIGER/Line only (auto-downloaded)
python scripts/generate_region1_boundary.py

# Use pre-downloaded TIGER/Line shapefiles to avoid re-downloading
python scripts/generate_region1_boundary.py --source-dir data/boundary/sources

# Specify custom output directory
python scripts/generate_region1_boundary.py --output-dir data/boundary
```

### Generate Interactive Map

```bash
# Generate map from existing boundary files
python scripts/write_boundary_map.py

# Specify custom input/output directories
python scripts/write_boundary_map.py \
    --boundary-dir data/boundary \
    --output output/boundary_map.html
```

### Full Regeneration Workflow

```bash
# 1. Generate ZIP code boundaries (requires zipcodes_orwa.csv)
python scripts/generate_zipcodes_orwa.py

# 2. Generate region boundary and registry
python scripts/generate_region1_boundary.py

# 3. Generate interactive map
python scripts/write_boundary_map.py
```

## Data Sources

### Census TIGER/Line

- **URL**: https://www2.census.gov/geo/tiger/
- **Coverage**: Full US extent
- **Resolution**: State and ZCTA (ZIP Code Tabulation Area) boundaries
- **License**: Public domain
- **Vintage**: 2023 (current)

### Oregon Metro RLIS

- **URL**: https://rlisdiscovery.oregonmetro.gov
- **Coverage**: Portland metro area (Clackamas, Multnomah, Washington counties)
- **Resolution**: Tax-lot accurate
- **License**: Creative Commons Attribution 4.0
- **Vintage**: Current (updated regularly)

### OpenDataSoft US ZIP Boundaries

- **URL**: https://public.opendatasoft.com/explore/dataset/us-zip-code-territory/
- **Coverage**: Full US extent
- **Resolution**: ZIP code boundaries
- **License**: Creative Commons Attribution 4.0
- **Vintage**: Current
