## Task 3: Processors

- [ ] 3.1 `src/processors/clip_to_boundary.py` ‚Äî load the OR/WA state boundary shapefile, filter to the polygon(s) for the current region, use `rasterio.mask.mask` to clip a raster array to the boundary; return the clipped array and updated transform; log clipped pixel dimensions and CRS; write `output/microclimate/geojson/proc_clipped_boundary.geojson` ‚Äî the OR/WA boundary Polygon itself as a GeoJSON feature where the popup shows: **Stage**: Processor 3.1 ‚Äî Clip to Boundary; **Inputs**: boundary shapefile path, `region_code`; **Outputs**: `crs`, `pixel_width_m`, `pixel_height_m`, `clipped_rows`, `clipped_cols`, `masked_pixel_count`
- [ ] 3.2 `src/processors/downscale.py` ‚Äî implement `reproject_to_lidar_grid(src_array, src_transform, src_crs, lidar_transform, lidar_crs, lidar_shape)` using `rasterio.warp.reproject` with `Resampling.bilinear`; all coarse rasters (PRISM, NLCD, NREL wind, Landsat LST) are passed through this function to produce arrays with the same shape as the LiDAR DEM; write `output/microclimate/geojson/proc_downscaled.geojson` ‚Äî district Polygons where each popup shows: **Stage**: Processor 3.2 ‚Äî Downscale; **Inputs**: source CRS, source resolution for each raster (PRISM 800 m, NLCD 30 m, NREL 2 km, Landsat 30 m), resampling method `bilinear`; **Outputs**: `target_crs`, `target_resolution_m`, `prism_hdd_downscaled`, `nlcd_impervious_downscaled`, `nrel_wind_10m_downscaled` (district means after reprojection)
- [ ] 3.3 `src/processors/terrain_analysis.py` ‚Äî compute: (a) aspect in degrees 0‚Äì360¬∞ from DEM gradient using `numpy.arctan2`; (b) slope in degrees from gradient magnitude; (c) TPI as pixel elevation minus mean elevation in annulus of inner radius 300 m and outer radius 1,000 m (use `scipy.ndimage` uniform filter on two radii and subtract); (d) wind shadow mask as binary array where TPI < 0 and aspect within ¬±90¬∞ of `PREVAILING_WIND_DEG + 180¬∞`; (e) lapse rate HDD addition as `(elevation_array ‚àí station_elevation_ft) / 1000 √ó LAPSE_RATE_HDD_PER_1000FT`; write `output/microclimate/geojson/proc_terrain_analysis.geojson` ‚Äî district Polygons where each popup shows: **Stage**: Processor 3.3 ‚Äî Terrain Analysis; **Inputs**: `LIDAR_DEM_RASTER`, TPI annulus inner `300 m` / outer `1000 m`, `PREVAILING_WIND_DEG`, `LAPSE_RATE_HDD_PER_1000FT`, `station_elevation_ft`; **Outputs**: `dominant_aspect_deg`, `mean_slope_deg`, `mean_tpi`, `terrain_position` (windward/leeward/valley/ridge), `wind_shadow_pct` (% of district pixels in wind shadow), `lapse_hdd_addition`
- [ ] 3.4 `src/processors/thermal_logic.py` ‚Äî compute: (a) `surface_albedo = 0.20 ‚àí (impervious / 100) √ó 0.15`; (b) solar aspect multiplier (0.8 for north-facing, 1.2 for south-facing, linear interpolation for other aspects); (c) `uhi_offset_f = (0.20 ‚àí surface_albedo) √ó SOLAR_IRRADIANCE_WM2 / 5.5 √ó 9/5`; (d) if Landsat LST is available, calibrate `uhi_offset_f` using a 70/30 weighted blend toward the observed urban‚Äìrural LST difference; log a warning if the Landsat-observed difference exceeds the modeled offset by more than 1.5¬∞C; write `output/microclimate/geojson/proc_thermal_logic.geojson` ‚Äî district Polygons where each popup shows: **Stage**: Processor 3.4 ‚Äî Thermal Logic; **Inputs**: `mean_impervious_pct`, `SOLAR_IRRADIANCE_WM2 = 200`, `dominant_aspect_deg`, Landsat LST available (yes/no); **Outputs**: `surface_albedo`, `solar_aspect_mult`, `uhi_offset_f` (pre-calibration), `uhi_offset_f_calibrated`, `uhi_hdd_reduction`, `landsat_urban_rural_diff_c`, `calibration_applied` (boolean)
- [ ] 3.5 `src/processors/wind_steering.py` ‚Äî first, construct a merged wind speed surface by combining the NREL 10 m wind grid (from loader 2.5, already downscaled to 1 m) with MesoWest station observations (from loader 2.4): at each station location, compute the difference between the observed mean wind speed and the NREL grid value, spatially interpolate those differences across the grid using `scipy.interpolate.griddata` (linear), and add the interpolated correction to the NREL surface to produce the final `wind_speed_surface` array; then compute stagnation multiplier: apply 0.7√ó to UHI offset where wind > 5 m/s and not in wind shadow; apply 1.3√ó where wind < 3 m/s and in wind shadow; apply 1.0√ó otherwise; compute wind infiltration multiplier as `1.0 + max(0, mean_wind_ms ‚àí 3) √ó 0.015`; apply Gorge floor of 1.15 for Columbia Gorge corridor districts; write `output/microclimate/geojson/proc_wind_steering.geojson` ‚Äî district Polygons where each popup shows: **Stage**: Processor 3.5 ‚Äî Wind Steering; **Inputs**: `nrel_wind_10m_ms` (raw grid), MesoWest station count used for correction, `wind_shadow_pct`, stagnation thresholds (< 3 m/s ‚Üí 1.3√ó, 3‚Äì5 m/s ‚Üí 1.0√ó, > 5 m/s ‚Üí 0.7√ó), infiltration formula `1.0 + max(0, wind ‚àí 3) √ó 0.015`, Gorge floor `1.15`; **Outputs**: `mean_wind_ms` (merged surface), `wind_correction_ms` (mean MesoWest adjustment applied), `wind_stagnation_mult`, `wind_infiltration_mult`, `gorge_floor_applied` (boolean), `uhi_offset_f_wind_adjusted`
- [ ] 3.6 `src/processors/anthropogenic_load.py` ‚Äî buffer road segments by AADT tier (50 m for AADT < 10,000; 100 m for 10,000‚Äì50,000; 200 m for > 50,000) using `geopandas.GeoDataFrame.buffer`; rasterize the buffered heat flux onto the 1 m grid using `rasterio.features.rasterize`; sum overlapping buffers; compute `road_temp_offset_f = road_heat_flux_wm2 / 5.5 √ó 9/5`; write two GeoJSON files: (a) `output/microclimate/geojson/proc_road_buffers.geojson` ‚Äî the buffered road Polygon features with popup showing: **Stage**: Processor 3.6 ‚Äî Road Buffers; **Inputs**: `AADT`, buffer tier applied, segment length; **Outputs**: `buffer_m`, `heat_flux_wm2`, `road_temp_offset_f`; (b) `output/microclimate/geojson/proc_anthropogenic_load.geojson` ‚Äî district Polygons where each popup shows: **Stage**: Processor 3.6 ‚Äî Anthropogenic Load (district aggregate); **Inputs**: road segment count within district, total buffered area; **Outputs**: `mean_road_heat_flux_wm2`, `road_temp_offset_f`, `road_segment_count`
- [ ] 3.7 `src/processors/combine_corrections.py` ‚Äî implement `compute_effective_hdd(base_hdd, terrain_mult, elev_addition, uhi_reduction, traffic_reduction)` returning `base_hdd √ó terrain_mult + elev_addition ‚àí uhi_reduction ‚àí traffic_reduction`; all inputs are numpy arrays of the same shape; return the `effective_hdd` array; write `output/microclimate/geojson/proc_effective_hdd_grid.geojson` ‚Äî district Polygons where each popup shows: **Stage**: Processor 3.7 ‚Äî Combine Corrections; **Inputs**: `base_hdd`, `terrain_mult`, `elev_addition`, `uhi_reduction`, `traffic_reduction`; **Outputs**: `effective_hdd`, and a breakdown line showing the full formula: `base_hdd √ó terrain_mult + elev_addition ‚àí uhi_reduction ‚àí traffic_reduction = effective_hdd`
- [ ] 3.8 `src/processors/weather_adjustment.py` ‚Äî implement `apply_weather_adjustment(effective_hdd_array, district_station_map, actual_station_hdd, normal_station_hdd)` that computes a per-station adjustment factor `adjustment = actual_station_hdd / normal_station_hdd` and applies `effective_hdd_adjusted = effective_hdd √ó adjustment` for each pixel based on its assigned base station; this step is only called when `--weather-year` is provided; if called, both `effective_hdd` (unadjusted) and `effective_hdd_adjusted` are retained; write `output/microclimate/geojson/proc_weather_adjustment.geojson` ‚Äî district Polygons where each popup shows: **Stage**: Processor 3.8 ‚Äî Weather Adjustment; **Inputs**: `weather_year`, `base_station`, `actual_station_hdd`, `normal_station_hdd`, `adjustment_factor`; **Outputs**: `effective_hdd` (unadjusted), `effective_hdd_adjusted`; if `--weather-year` is not specified, this GeoJSON is not written
